## Project Overview
This project focuses on improving the student portal by fixing common usability and performance problems reported by students. Based on survey data collected from students in our CSS class, our team identified repeated issues with login, navigation, slow loading, and broken pages.

Later, we expanded the idea by adding an AI assistant to the student portal. The goal is to help students find resources faster, understand where to go, and get support more easily.

This repository shows the system story, the proposed architecture, and my role as a Software Designer.

## Problem Being Solved
Students have trouble using the portal for basic tasks. The most common issues found in the survey were:

- difficulty accessing schedules
- difficulty finding administrative services
- slow loading times
- confusing navigation
- pages that fail to load

These problems make routine academic tasks harder and more frustrating.

## Core Problem
Students experience friction during login and navigation, which reduces efficiency and makes the portal harder to use.

## System Vision
Our goal is to create a student portal that is easier to use, more reliable, and faster. The improved system should:

- improve navigation clarity
- reduce loading times
- provide working and consistent pages
- make academic resources easier to access
- help students get support faster

## Proposed Solution
The proposed solution is a redesigned student portal with a built-in AI assistant.

The portal remains the main place where students log in, find information, and access services. The AI assistant helps students by answering common questions, guiding them to the right page, and helping them understand what to do next.

Behind the portal, Odoo can be used to manage requests, workflows, and internal service handling.

## System Requirements
The main system requirements are:

- AI assistant integration to help students find resources
- backend improvements to reduce loading time
- fixes for broken or non-loading pages
- better navigation across services
- more consistent page design and workflow structure

## Main Users
The main users of this portal are students.

This version of the portal is designed for the student experience. Staff may use the backend side to process requests, but they are not the primary users. Professor-facing functions would belong to a separate version or module.
